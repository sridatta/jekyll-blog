    H@B Backend Web Workshop

    by Sridatta Thatipamala

    How does the Web work?

    The Client

    Web clients make requests to web servers. A client can be:

    • a Web browser
    • a command line utility (curl, WGET)
    • another server

    Most important component of a request: the URL

    The Server

    Web servers accept connections from clients. When a client sends a request, the server responds with:

    • A status code

      200 OK, the famous 404 Not Found, etc.

    • A response body

      The actual page data

    Your job as a backend coder is to write code that gets called when certain URLs are requested, retrieve user-supplied data (if necessary), and respond with the appropriate data.
